Aprenda Requisitos Brincando

Navegue pelas diferentes abas de estudo para dominar Requisitos Funcionais e Não Funcionais.

Requisitos Funcionais (RF)

Descrevem as funcionalidades concretas e serviços que o sistema deve fornecer aos usuários.

  • Sistema deve permitir o cadastro de usuários.
  • O usuário deve poder adicionar itens ao carrinho.

Requisitos Não Funcionais (RNF)

Descrevem atributos de qualidade, restrições e características globais do sistema.

  • O software deve carregar em menos de 2 segundos.
  • Criptografia completa do banco de dados para segurança.

Padrão de Qualidade: ISO/IEC 25010

Em Engenharia de Software, as características dos sistemas (especialmente os Requisitos Não Funcionais) são agrupadas e norteadas pelo modelo de qualidade internacional ISO/IEC 25010. Abaixo estão os 8 pilares principais:

Adequação Funcional

O grau com que o sistema satisfaz as necessidades declaradas sob condições especificadas (Cobertura, Correção e Adequação às tarefas).

Eficiência de Desempenho

Desempenho relativo à otimização da quantidade de recursos e tempo utilizados em condições ideais.

Compatibilidade

Capacidade do produto de trocar informações com outros sistemas e compartilhar o mesmo ambiente sistêmico sem quebra.

Usabilidade

Facilidade com que o sistema pode ser aprendido, operado e compreendido por usuários, incluindo sua Acessibilidade visual (UI) e UX.

Confiabilidade

Frequência logada com que um sistema atinge o sucesso e opera sem falhas em um tempo específico de disponibilidade combinada.

Segurança

Proteção rigorosa de informações para que os dados sensíveis sejam transmitidos apenas para sistemas ou pessoas autenticadas.

Manutenibilidade

Facilidade do código ser lido para que um software seja alterado, atualizado e testado prevenindo novos bugs.

Portabilidade

Eficácia, coesão estrutural e flexibilidade tecnológica em transferir a infraestrutura e instalação do sistema para outro ambiente de produção.
